Lua 1: Beginnings

From humble beginnings...

Lua is a programming language. Each program is made up of instructions.
When a program is run, instructions start executing from top to bottom.
Our first instruction will be creating variables.

Think of variables like boxes. They can hold anything.
Our first instruction: set name on left equal to value on right.
x = 5
y = x
y = 2
We set x -> 5, then y -> (the value of x) -> 5, then y -> 2.
Lua has math operators, as listed in the table below.
Name Symbol Usage
Add + x = 1 + 10
Subtract - x = 1 - 10
Multiply * x = 1 * 10
Divide / x = 1 / 10
Parentheses can be used to group operations,
which normally follow PEMDAS order.
A boolean value is either true or false.
Relational operators take in inputs and return booleans.
Name Symbol Usage
Greater than > x = (1 > 10) -- x is false
Less than < x = 1 < 10
Equal == x = 1 == 10
Not equal ~= x = 1 ~= 10
Greater than or equal >= x = 1 >= 10
Less than or equal <= x = 1 <= 10
Lua has one more "fundamental" value: nil.
The value "nil" represents nothingness, or lack of existence.
"nil" and "false" are not the same.
Lua has comments, which are informative instructions that have no effect on code.
--This is a comment
x = 5 --Set x to 5.
--[[This is  
a multi-line comment.
Code anywhere between these
brackets does not execute.]]
Let's introduce "blocks". Blocks can be thought of as a section of code.
Sometimes code should execute if some condition(s) are valid.
For this, Lua has the "if block".
--[[
if boolean_expression then 
	do code within the block once provided boolean_expression == true
end --end the block
]]

x = 4
if x == 5 then
	y = 6 --Note the tab here
end
y = 7 
--x is 4, y is 7

if true then
	y = 8 --And also, the tab here
end
--x is still 4, y is 8
Generally, code within a block is always tabbed. 
if true then --This whole block executes.
  --This block below does not execute.
	--We are "nesting" this if block within the first if block.
	if false then
		if x == 5 then
			y = 6
		end
	end
	--End of block.
	x = 8 --This executes.
end
Note that for the value "nil", 
y = nil
if nil then
	y = 5
end
if y then
	y = 6
end
--y will still be nil, neither if block executes
Likewise, comparable to add and subtract, Lua has operators for booleans.
Name Symbol Usage
AND (true if both inputs are true) and x = (true and false)
OR (true if one or more input is true) or x = true or false
NOT (negates, reverses) not x = not (not true) -- == not false == true
Functions are a new block that we can already have or define as new.
Once we have it, we can call it to actually execute it.
Functions take in inputs (also called arguments, parameters), and can return.
Return is another instruction we'll define as a "result" of a function.
--[[
function name_here(input1, input2, ...)
	execute a set of instructions
	optional: return (we can return only once)
end

name_here(1, 2, ...)
]]

function communicate(number)
	print(number) --print is a function that takes in anything and shows it to screen
	print(number * 2) --print DOES NOT return. Printing ~= Returning
	print(number == number + 4) --note that addition is done first, then the equal == operator
end

communicate(8)
Here we must talk about "type". So far, we've seen these types:
number, boolean, nil. Variables can be of any type, Lua is free.
However, some functions act as if an input is of a certain type.
function add(a,b) --Only works on numbers!
	return a + b --This is the "result" of the function.
end
x = add(3,5) --x is set to the result of add(3,5), which is 8
y = add(true,false) --not valid! Lua does not define true added to false
We'll talk about type later. Lua is very loose with this.
Other languages have strict types, which means that variables must be of a certain type.
Lua trusts you with this.

Let's define another type that's been missing: the string.
A string is simply a collection of letters.
It can represent a word, text, or any collection of symbols.
stringy = "abc"
other_stringy = "def"
There are many operations we can do on strings, and for now,
we'll only define one: combining strings, through the operator ".."
stringy = "c"
other_stringy = "s"
combined = stringy .. other_stringy --"cs"
x = 61
other_combined = combined .. x .. "a" 
print(other_combined) --"cs61a"
Our last topic is tables. Tables can be thought as an apartment.
In Lua, the addresses can be anything: numbers, booleans, even functions.
We access an address (or an "index") and we get the occupant of the room.
Let's define the following syntax:
x = {} --Create an empty table
x[1] = "stringy"
x[2] = 61
x[1] = x[1] + x[2] --Note that this will set the NEW value of x[1] to the OLD value of x[1] and x[2]
print(x[3]) --nil --This does not error.

--These two lines are equivalent. Why?
x[x[1]] = "another_stringy"  
--x["stringy"] = "another_stringy"

x.stringy = x[1] --Same as x["stringy"] = x[1]
print(x[1])
In this way, we can think of a table as a function,
where the index is the only input, and the result is the data located at the index.
